# Contributing to HelioCast 🌟

We love your input! We want to make contributing to HelioCast as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features

## Our Development Process

We use GitHub to track issues and pull requests. All changes should be submitted via Pull Requests.

1. **Fork** the repo and create your branch from `main`.
2. **Setup your environment** (run `npm install` and verify the server starts).
3. **Make your changes**. If you added code that should be tested, add tests.
4. **Build test**: Ensure `npm run build` runs successfully without warnings.
5. **Issue a Pull Request** describing your changes.
